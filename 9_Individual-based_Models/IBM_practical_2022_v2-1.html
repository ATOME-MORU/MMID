Implementing transmission

For each individual, the probability of infection is a reflection of his/hers susceptibility and the number of infectious contacts he/she receives each timestep. Let’s say the number of potentially infectious contacts is given by parameter beta.

Create parameter beta and assign it a value between 2 and 5. If I have 1 contact per day, the chances that I will become infected is given by the probability that that 1 person I have contract with is presently infectious.

So, let’s create a variable lambda to represent the probability that a contact occurs with a presently infectious individual:

beta <- 3
lambda <- beta.sum(pop==2)/length(pop)

Now let’s make it realistic by making it stochastic. In programming terms that means assigning some randomness to the contacts between individuals to assess whether or not an infection takes place. We start by casting a uniformly distributed random number for each individual.

random.no <- runif(length(pop))

So, let’s implement the transmission process at the individual level. For this we need another ‘for’ loop.

Let’s look at individuals one by one

for(i in 1:length(pop))
{
#Simulate what happens to each individual
}

Remember that these processes are all happening in a single timestep. So this second ‘for’ loop has to be within the first ‘for’ loop. Also note the difference in loop lengths. If we take lambda to literally be the probability to become infectious in each timestep, we simply have to test whether the random number created for each individual is less than lambda. Obviously, you need to check the susceptibility condition. If an individual is susceptible (ie. state 0), we’ll compare its corresponding random no. with lambda. If it’s less than lambda, the individual gets infected and we change that individual’s state to 1 (first week of infection).

if(pop[i]==0)
{
      if(random.no[i]<lambda)
{
        pop[i]<- 1
      }
}

Let’s say each timestep is equivalent to one week and that both latent and infectious periods are also one week. If individuals are latent or infectious, we’ll just move them on to the next state each timestep. Since we’ve coded the states S, E, I and R numerically as 0, 1, 2, and 3 respective, we just have to add 1 to move them to the next state.

else if(pop[i]==1 | pop[i]==2)
{
      pop[i] <- pop[i] + 1
}

That’s it. You’ve just written your first individual-based model in R! The full code:

initS <- 19
initI <- 1
beta <- 1.5
gamma<-1
tau<-1
no.of.timesteps <- 10
pop <- c(rep(0,initS),rep(1,initI))
sim.table <- as.data.frame(matrix(NA, no.of.timesteps, 5))
colnames(sim.table) <- c('WeekNo','S','E','I','R')

for(t in 1:no.of.timesteps)
{
  #Simulate what happens at each timestep
  sim.table$WeekNo[t] <- t
  sim.table$S[t]  <- sum(pop==0)
  sim.table$E[t] <- sum(pop==1)
  sim.table$I[t] <- sum(pop==2)
  sim.table$R[t]  <- sum(pop==3)
  lambda <- beta*sum(pop==1| pop==2)/length(pop)
  random.no <- runif(length(pop))
  
  for(i in 1:length(pop))
  {
    #Simulate what happens to each individual
    if(pop[i]==0)
    {
      if(random.no[i]<lambda)
      {
        pop[i] <- 1
      }
    }
    else if(pop[i]==1 | pop[i]==2)
    {
      pop[i] <- pop[i] + 1
    }
    
  }
  
}

A more general framework including recovery and immunity loss

Let’s implement latency, recovery from infection and no immunity as stochastic processes to better reflect malaria epidemiology.

As before, let’s start by generating a vector pop of individuals and an output table sim.table.

initS <- 19
initI <- 1
beta <- 0.5
no.of.timesteps <- 100
pop <- c(rep(0,initS),rep(1,initI))
sim.table <- as.data.frame(matrix(NA, no.of.timesteps, 5))
colnames(sim.table) <- c('Day','S','E','I','R')

Notice the change in number of timesteps. We are now using a daily timestep. Write the new function for the daily probability of infection lambda:

for(t in 1:no.of.timesteps)
{
  #Simulate what happens at each timestep
  sim.table$Day[t] <- t
  sim.table$S[t] <- sum(pop==0)
  sim.table$E[t] <- sum(pop==1)
  sim.table$I[t] <- sum(pop==2)
  sim.table$R[t] <- sum(pop==3)
  lambda <- beta*sum(pop==2)/length(pop)
    
}

We now have 3 state transitions: Infection, Latency and Recovery. Define parameters for the daily probabilities of becoming infectious (gamma) and recovering from infection (tau), assuming latency lasts an average of 5 days and people are usually infectious for 10 days.

 {

A more general contact structure

Now, let’s explore a different conceptualization of the force of infection for an individual based model. In the example above, all individuals had an identical daily risk of infection given by lambda. In the real world, individuals self-organize into complex contact networks. The igraph R library (https://igraph.org/r/doc/plot.common.html) contains a lot of useful resources to generate and plot networks. Here is an example of how to generate a small world type network in R.

library(igraph)
init_S <- 100
init_I <- 10
pop <- c(rep(0, init_S), rep(1, init_I))
connectivity <- 2
rewiring <- 0.15
nw <- sample_smallworld(1, length(pop), connectivity, rewiring)
plot(nw)

We can easily access the list of individual links:

head(as_edgelist(nw, names = TRUE))
##      [,1] [,2]
## [1,]    1    2
## [2,]    2    3
## [3,]    3    4
## [4,]    4    5
## [5,]    5    6
## [6,]    6    7

Where each row establishes a link (edge) between the two individuals (vertices) on the graph. To find the neighbors (vertices it is linked to) of a specific individual on that network, we can simply write:

neighborhood(nw, nodes = 1:3)
## [[1]]
## + 5/110 vertices, from 6d4e0d0:
## [1]   1   2   3 109 110
## 
## [[2]]
## + 5/110 vertices, from 6d4e0d0:
## [1]  2  1  3  4 19
## 
## [[3]]
## + 6/110 vertices, from 6d4e0d0:
## [1]  3  1  2  4  5 15

In this case we are looking at the neighbors for the first 3 vertices. The first person is linked to individuals 2, 3, 61, 109, and 110 – you can always ignore the first number as it is the link with itself.

Let’s take the explicit meaning of beta – the number of mean infectious contacts per person per day. For each infectious person, we can then sample a random number of contacts to be had every day given a certain distribution. For a Poisson distribution that would look like:

ncontacts <- rpois(1, beta)

So if you draw 10 numbers from that distribution with a beta of 1, you would get, e.g:

rpois(10,1)
##  [1] 3 0 1 0 2 1 0 1 0 0

In which case 2 individuals would have no contacts that day, 2 would have 1 contact, and 6 would have 2 contacts. As you can see, stochasticity will play a large role here. One we know how many potentially infectious contacts per day each individual engages in, we need to know who those contacts where with to check whether a new infection occurs. Using the neighborhood function explored about it is straightforward to sample X individuals from the list of possible contacts for individual i, using the sample function. Note that we need to remove the first entry which is the self-link by doing [-1] on that vector.

pcontacted <- sample(as.vector(neighborhood(nw, nodes = i)[[1]][-1]), ncontacts, replace = TRUE)

A more realistic use case of informing networks from proxy data

Unless one can monitor all individual interactions for a large enough group of people, informing an individual based network in detail is nigh impossible. Several efforts have been made to characterize contact networks in different social contexts, resulting in macro-level abstractions that can be directly used as model inputs. The example above sets the network of (potentially infectious) contacts as a small world type network. In your experience as a researcher, you are likely to find gold standard contact matrices such as the ones used for respiratory infectious diseases, or mobility data in some form or another (from mobile phone usage records, travel surveys, etc). The data usually come in the form of a square matrix (except for individual level data like mobile phone records), which essentially stands as a proximity matrix between places/areas/groups. One of the most used contact matrices over recent years is the Prem K, Cook AR, Jit M (2017) Projecting social contact matrices in 152 countries using contact surveys and demographic data age contact dataset. Lets import the UK contact matrices to R.

## import contact matrices
c_home <- as.matrix(read.table("contact_home.txt", sep = ",", header = F))
c_school <- as.matrix(read.table("contact_school.txt", sep = ",", header = F))
c_work <- as.matrix(read.table("contact_work.txt", sep = ",", header = F))
c_other <- as.matrix(read.table("contact_other.txt", sep = ",", header = F))
total_contacts <- c_home + c_school + c_other

You can use the plot.matrix library to quickly visualize the proximity between age groups.

library(plot.matrix)
plot(total_contacts)

Clearly, this is a strong diagonal matrix, representing high umber of contacts between people of the same age group, alongside some off-diagonal contacts which likely represent inter-generational contacts within families (children and their parents). Given the information in this matrix we could then inform a strongly age-dependent contact structure in our infection function. The first step is to assign each individual an age group. For that we use populations numbers (in thousands) from the UN 2019 Revision of World Population Prospects.

## import age structure
popstruc <- read.csv("pop_structure_X.csv", header = T)
# convert from 1000s to total numbers
popstruc[, 2] <- popstruc[, 2] * 1000

## assign an age group to each individual
samplesize <- length(pop)
ages <- with(popstruc, sample(length(popstruc[, 1]),
    samplesize, p = (popstruc[, 2] / sum(popstruc[, 2])), replace = T))

We can then create our synthetic population contact network according to the adjacency matrix above.

## build network based on age
X <- as.matrix(total_contacts)
g <- graph_from_adjacency_matrix(X, mode = "lower", 
    weighted = "weight", diag = T)
g2 <- delete.edges(g, which(E(g)$weight < 1))
plot(g2, edge.width = E(g2)$weight)

Note that I’ve deleted all contacts between groups with a weight lower than 1, meaning less than one contact, on average, between individuals in those groups per day. After initialising infections in individuals from random age groups, we can investigate how to write the code for our infection function.

## init_Ialise infections in random individuals
starting_infs <- sample(length(pop), init_I)
pop[starting_infs] <- 1

Conceptually, what we need to accomplish is to write a block of code that for each infected person:

  • draws a random number of potentially infectious contacts per day (ncontacts) according to our contact rate and probability that each contact is infectious.

  • allocates those contacts to people of other age groups according to the distributions given by the total_contacts matrix.

## INFECTION ##
        if (pop[i] == 2) {
            # how many infectious contacts do you have today
            ncontacts <- rpois(1, p * sum(total_contacts[ages[i], ]))
            # print(ncontacts)
            if (ncontacts > 0) { 
                for (w in 1:ncontacts) {
                    # which age group do you get in contact with
                    target_age <- sample(length(total_contacts[, 1]), 1,
                    prob = total_contacts[ages[i], ], replace = TRUE)
                    target_person <- sample(which(ages == target_age), 1)
                    if (pop[target_person] == 0) {
                        pop[target_person] <- 1
                    }
                }
            }
        }
